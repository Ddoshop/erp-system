# 📋 Жизненный цикл тендера в ERP системе

## Обзор процесса

Тендер проходит через 4 основных статуса и 5 рабочих этапов с участием нескольких ролей пользователей.

```
┌─────────────────────────────────────────────────────────────────┐
│                    СОЗДАНИЕ ТЕНДЕРА                             │
│                                                                  │
│  Менеджер/Админ нажимает кнопку "+ Тендер"                     │
│                                                                  │
│  ┌─── Режим ──────────────────────────────────────────────┐    │
│  │  A) Вручную — заполнить форму самостоятельно           │    │

│  данные заполняются вручную в форме проверки.                   │

│  Тендер создаётся в статусе "Прием заявок" (по умолчанию)      │

### Режим B — Импорт по ID закупки (рекомендуемый)

Менеджер вставляет **noticeInfoId** (или ссылку целиком):

```
Например: 19515624
Или: https://zakupki.gov.ru/epz/order/notice/notice223/common-info.html?noticeInfoId=19515624
```

#### Что делает система:
1. Извлекает числовой ID из введённого значения (regex)
2. Формирует два URL по шаблону:
   - `common-info.html?noticeInfoId={id}` — основные данные тендера
   - `documents.html?noticeInfoId={id}` — список прикреплённых документов
3. Пытается скачать обе страницы (таймаут 8 сек каждая)
4. Парсит HTML и заполняет все доступные поля
5. Возвращает `{ success: true, partial: false, data: {...} }`

#### При недоступности gov.ru:
- Fetch возвращает `null` без исключения
- Возвращается `{ success: true, partial: true, data: { number: "ID", ... } }`
- В браузере открывается форма с заголовком **"Сайт недоступен — заполните вручную"**

#### Форма проверки данных (оба режима):
```javascript
showForm(partial ? "Сайт недоступен — заполните вручную" : "Проверьте данные перед сохранением", [
  { name: "number",          label: "Номер закупки",        required: true },
  { name: "registry_number", label: "Реестровый номер" },

### Документы тендера (`tender_files`)
При успешном импорте система сохраняет список документов в таблицу `tender_files` и пытается скачать их локально в `data/tender_files/{tender_id}/`. Файлы доступны через `/files/{tender_id}/{filename}`.

| Поле | Описание |
|------|---------|
| `source_url` | Оригинальная ссылка на gov.ru |
| `local_url` | Локальный путь `/files/...` (если скачан) |
| `file_name` | Имя файла |
| `file_ext` | Расширение (pdf / docx / xlsx / ...) |
| `doc_type` | Тип документа (ТЗ, Расчёт, Проект контракта, ...) |
```

### Возможные начальные статусы
- **"open"** (Прием заявок) — тендер опубликован, идёт приём заявок **[по умолчанию]**
- **"draft"** (Черновик) — тендер не опубликован, не виден поставщикам

## 2️⃣ ЭТАП ПУБЛИКАЦИИ

### Действующие лица
- **Менеджер** / **Администратор** — публикатор тендера

### Переход статуса
**Draft → Open** (Черновик → Прием заявок)

### Как это происходит
Менеджер нажимает кнопку "Изменить" на строке тендера и меняет статус через модальное окно:

```javascript
showForm("Редактировать тендер", [
  // ... все поля тендера
  { name: "status", label: "Статус", type: "select", 
    options: [
      { value: "open",   label: "Прием заявок" },
      { value: "review", label: "На рассмотрении" },
      { value: "draft",  label: "Черновик" },
      { value: "closed", label: "Закрыт" },
    ], 
    value: "open" // меняется здесь
  },
])
```

### Действия системы при публикации
1. ✅ Тендер становится видимым на странице "Тендеры" для поставщиков (роль `supplier`)
2. ✅ Кнопка "Подать заявку" становится активной для всех поставщиков
3. ✅ Включается таймер приёма заявок (обычно до `deadline`)
4. ✅ Начинается сбор коммерческих предложений (quotations)

### API запрос
```http
PUT /api/tenders/{id}
Content-Type: application/json

{
  "number": "44-02-2026-001",
  "lot": "Закупка офисной бумаги А4",
  "client": "ООО Госзаказ",
  "price": 500000,
  "deadline": "2026-04-15",
  "status": "open"  // ← ИЗМЕНЕНО
}
```

## 2.5️⃣ ЭТАП ПОДБОРА ТОВАРОВ (СНАБЖЕНИЕ)

### Действующие лица
- **Снабженец** (supply) — просматривает документацию и подбирает товарные позиции
- **Менеджер** / **Администратор** — также имеют доступ к этому разделу

### Триггер
Менеджер/Снабженец нажимает кнопку **"Подбор товаров"** в строке тендера (видна для ролей `supply`, `manager`, `admin`).

### Что видит снабженец в панели
1. **Документация** — список файлов тендера (ТЗ, расчёты, проект контракта и т.д.) с кликабельными ссылками
2. **Подобранные товары** — таблица позиций (кто добавил, когда, ориентировочная сумма)
3. Кнопка **"+ Добавить позицию"**

### Форма добавления товарной позиции
```javascript
showForm("Добавить товарную позицию", [
  { name: "article",   label: "Артикул / Код",                placeholder: "Необязательно" },
  { name: "name",      label: "Наименование товара",          required: true },
  { name: "quantity",  label: "Количество",                   type: "number", required: true },
  { name: "unit",      label: "Единица измерения",            value: "шт" },   // шт / кг / л / м / уп
  { name: "price_est", label: "Ориентировочная цена (руб.)", type: "number" },
  { name: "note",      label: "Примечание",                   type: "textarea" },
])
```

### Таблица `tender_items`
| Поле | Тип | Описание |
|------|-----|---------|
| `tender_id` | FK | Ссылка на тендер |
| `user_id` | FK | Кто добавил позицию |
| `article` | string | Артикул/код товара |
| `name` | string | Наименование товара |
| `quantity` | real | Количество |
| `unit` | string | Ед. измерения (шт, кг, л, м, уп) |
| `price_est` | real | Ориентировочная цена за единицу |
| `note` | text | Примечание |

### API
```http
GET    /api/tenders/:id/items          # список позиций (все авторизованные)
POST   /api/tenders/:id/items          # добавить (supply / manager / admin)
DELETE /api/tenders/:id/items/:itemId  # удалить (supply / manager / admin)
```

### Итог панели
Снабженец видит суммарное количество позиций и **ориентировочную сумму** (сумма `price_est × quantity` по всем позициям).

### Учётные данные снабженца (dev)
| Поле | Значение |
|------|---------|
| Email | `supply@technotrade.ru` |
| Пароль | `123456` |
| Роль | supply / Снабженец |

#### API запрос
```http

  "supplier_name": "ООО Снабсервис",
  "supplier_email": "sales@snab.ru",

### Расчёт НМЦК

```javascript
// Метод сопоставимых рыночных цен (среднее арифметическое)
const nmck = quotes.reduce((sum, q) => sum + q.price, 0) / quotes.length;

// Отображение в панели:
// "Среднее (метод сопоставимых рыночных цен): 475000 ₽ — 3 КП"
```

#### Алгоритм
1. Собираются все КП по тендеру
2. Вычисляется среднее арифметическое цен
3. Это значение — **средняя НМЦК** (обоснованная стоимость)
4. Отображается для информации менеджеру

## 4️⃣ ЭТАП ОЦЕНКИ И РАССМОТРЕНИЯ

### Действующие лица
- **Менеджер** / **Администратор** — оценивает и утверждает заявки

### Переход статуса
**Open → Review** (Прием заявок → На рассмотрении)

### Как переходить в Review
```javascript
const data = await showForm("Редактировать тендер", [
  // ... все прочие поля
  { name: "status", label: "Статус", type: "select", 
    options: TENDER_STATUSES, 
    value: "review"  // ← ИЗМЕНЯЕТСЯ ЗДЕСЬ
  },
])
```

### Действия менеджера на этапе Review

#### 1. Просмотр всех заявок поставщиков
Переход на страницу "/applications"

```
┌───────────────────────────────────────────────────┐
│  Заявки поставщиков                               │
├───────────────────────────────────────────────────┤
│ Тендер | Лот | Компания | Цена | Срок | Статус │
├───────────────────────────────────────────────────┤
│ 44-02-2026-001 | Бумага | ООО Снабсервис | ... │
│  [Одобрить] [Отклонить]                         │
├───────────────────────────────────────────────────┤

│  [Одобрить] [Отклонить]                         │
└───────────────────────────────────────────────────┘
```

#### 2. Утверждение/Отклонение заявок
```javascript

await api(`/api/applications/{id}/status`, "PUT", {

});

// Отклонение
if (await confirmModal("Отклонить заявку поставщика?", "Отклонить")) {
  await api(`/api/applications/{id}/status`, "PUT", { 
    status: "rejected" 
  });
}
```

#### 3. Финальное сравнение КП
По панели "КП / НМЦК" сравниваются все коммерческие предложения:

```

```

## 🗂️ СВЯЗАННЫЕ ТАБЛИЦЫ БД

```sql
-- Основные
TABLE tenders {
  id PK
  number VARCHAR
  lot VARCHAR
  client VARCHAR
  price DECIMAL
  deadline DATE
  status ENUM('draft', 'open', 'review', 'closed')
  created_at TIMESTAMP
}

-- Заявки поставщиков
TABLE applications {
  id PK
  tender_id FK → tenders.id
  applicant_id FK → users.id
  price DECIMAL
  delivery_days INT
  note TEXT
  status ENUM('pending', 'approved', 'rejected')
  created_at TIMESTAMP
}

-- Коммерческие предложения (КП)
TABLE price_quotes {
  id PK
  tender_id FK → tenders.id
  supplier_name VARCHAR
  supplier_email VARCHAR
  price DECIMAL
  delivery_days INT
  note TEXT
  created_at TIMESTAMP
}

-- Заказы (результат закрытия тендера)
TABLE orders {
  id PK
  tender_id FK → tenders.id
  supplier_id FK → applications.applicant_id
  amount DECIMAL
  status ENUM('draft', 'open', 'review', 'closed')
  created_at TIMESTAMP
}
```

### Модальная система
Все формы используют `showForm()` и `confirmModal()` вместо `prompt()` / `confirm()`:

// Создание
const data = await showForm("Добавить тендер", fields);

## 📊 ПРИМЕР ПОЛНОГО ЦИКЛА

### Сценарий: Закупка офисной бумаги

**1. Менеджер Иван создаёт тендер**
- Номер: "44-02-2026-001"
- Лот: "Закупка офисной бумаги А4"
- НМЦК: 500 000 ₽
- Срок: 2026-04-15
- Статус: DRAFT → меняет на OPEN

**2. Три поставщика подают заявки** (статус: pending)
- ООО Снабсервис: 480 000 ₽ + КП
- АО Универсал: 450 000 ₽ + КП
- ЗАО Логистик: 520 000 ₽ (без КП)

**3. Система рассчитывает НМЦК**
(480 000 + 450 000 + 520 000) / 3 = **483 333 ₽**

**4. Иван переводит в REVIEW и одобряет заявки**
- ✅ ООО Снабсервис (цена близка к НМЦК)
- ✅ АО Универсал (самая низкая цена)
- ❌ ЗАО Логистик (выше НМЦК, нет обоснования)

**5. Иван закрывает тендер (CLOSED)**
- Система создаёт заказ (order) на ООО Снабсервис (как основного)
- Отправляются письма:
  - Снабсервис: "Вы выбраны!"
  - Универсал: "Вы в очереди резерва"
  - Логистик: "Спасибо за участие"

**6. Переход к этапу выполнения заказа**
```
Тендер "44-02-2026-001" ──► Заказ #15 ──► Поставка ──► Закрытие
          (CLOSED)          (OPEN)        (PROGRESS)   (COMPLETED)
```
