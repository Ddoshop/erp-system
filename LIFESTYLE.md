# Жизненный цикл закупки (LIFESTYLE)

## 1. Общая схема

Путь закупки в системе:

1. Менеджер создает тендер (вручную или импортом с госзакупок).
2. Подборщик добавляет товарные позиции в тендер.
3. Менеджер формирует комплект заявки и подает заявку.
4. После дедлайна тендер переходит в работу комиссии.
5. Менеджер фиксирует решение комиссии: выиграли или проиграли.
6. Если выиграли: договор подписывается в 2 шага.
7. Подборщик создает заказы по позициям подписанного договора (можно частями, с ценой закупки до копеек).
8. Система автоматически создает счет к каждому заказу.
9. Бухгалтер отмечает оплату счета, заказ становится доступен для склада.
10. Подборщик переводит заказ в статус Поставлен на склад.
11. Бухгалтер формирует документы отгрузки и передает поставку логисту.
12. Логист проводит поставку до статуса Получено клиентом.
13. Бухгалтер подтверждает оплату по поставке и закрывает поставку.
14. Когда все поставки по тендеру закрыты, тендер автоматически получает статус Исполнен и уходит в архив.

---

## 2. Роли и зона ответственности

1. Менеджер:
- создание и редактирование тендеров;
- подача заявок;
- фиксация решения комиссии;
- подписание договора (инициация этапов).

2. Подборщик:
- наполнение тендера позициями;
- создание заказов по договору;
- перевод оплаченных заказов в Поставлен на склад.

3. Бухгалтер:
- контроль счетов и оплаты заказов;
- генерация документов по поставке (ACT/UPD/Invoice);
- передача поставки логисту;
- финальное закрытие поставки по оплате.

4. Логист:
- ведение этапов отгрузки: назначено -> отгружено -> получено.

5. Администратор:
- имеет расширенный доступ и может выполнять действия нескольких ролей.

---

## 3. Статусы тендера

### 3.1 Площадочный статус (status)

1. open - Прием заявок
2. commission - Работа комиссии
3. awaiting_signing - Ожидание подписания на площадке
4. signed - Подписан
5. executed - Исполнен
6. closed - Закрыт (используется для сценариев проигрыша/закрытия)

### 3.2 Внутренний статус (internal_status)

1. awaiting_picking - Ожидает подбора
2. awaiting_application - Ожидает подачи заявки
3. submitted - Заявка подана
4. won_waiting_sign - Выиграли, ожидает подписания
5. signed_ours - Подписан с нашей стороны
6. signed_both - Подписан с двух сторон
7. archived_lost - Проиграли (архив)
8. executed - Исполнен

---

## 4. Ветка проигрыша

1. Комиссия = lost.
2. Тендер переводится в archived_lost.
3. Тендер архивируется (is_archived = 1).
4. Дальнейшие шаги заказа/поставки не выполняются.

---

## 5. Ветка выигрыша

1. Комиссия = won.
2. Тендер переходит в won_waiting_sign и awaiting_signing.
3. Менеджер подписывает договор с нашей стороны -> signed_ours.
4. Менеджер подтверждает подписание с двух сторон -> signed_both, площадочный статус signed.
5. Тендер становится доступным для создания заказов по позициям.

---

## 6. Жизненный цикл заказа

1. Заказ создается по позициям тендера (частично или полностью).
2. При создании заказа автоматически создается счет (invoice) со статусом unpaid.
3. Статусы заказа:
- awaiting_payment - ожидает оплаты;
- paid - оплачен;
- stocked - поставлен на склад;
- closed - закрыт.

4. Переходы:
- бухгалтер отмечает счет paid -> заказ paid;
- подборщик переводит paid -> stocked;
- после закрытия поставки заказ автоматически -> closed.

5. Важное правило UI:
- после статуса stocked заказ исчезает из вкладки Заказы.

---

## 7. Жизненный цикл поставки

Поставка создается автоматически для заказа, когда заказ попадает в stocked.

1. Начальный статус поставки: warehouse.
2. Бухгалтер генерирует документы: ACT, UPD, Invoice.
3. Бухгалтер выполняет handover (transfer_ready = 1).
4. Логист ведет статусы:
- warehouse -> scheduled -> shipped -> received.

5. После received система ставит awaiting_payment.
6. Бухгалтер подтверждает оплату по поставке -> статус closed.

7. Важное правило UI:
- после статуса closed поставка исчезает из вкладки Поставки.

---

## 8. Авто-завершение тендера

Правило завершения работает автоматически:

1. Если у тендера есть заказы.
2. И все эти заказы имеют статус closed.
3. Тогда тендер автоматически переводится в:
- status = executed;
- internal_status = executed;
- is_archived = 1;
- archived_at = текущая дата.

Итог: полностью выполненная закупка уходит в архив как Исполнен.

---

## 9. Итоговый поток в одну строку

Тендер -> Подбор -> Подача -> Комиссия -> (Проигрыш в архив | Выигрыш -> Подписание x2 -> Заказы -> Счета -> Склад -> Поставка -> Оплата -> Закрытие) -> Исполнен -> Архив.
